## 1. Platform evaluation (artifacts in ../mergelearn-eval/)

- 54 cards judged across 5 runs by an independent, per-card fresh judge.
- Distribution (not just means): 13 good / 24 weak / 17 bad. ~24% good rate.
- Dimension correlations with overall: teaching_value 0.89, answerable 0.82,
  answer_correct 0.81. The two UNCHECKABLE dimensions (answer_correct 44% fail,
  answerable 41% fail) are the ceiling; the mechanically-gated dimensions
  (no_leak 7% fail, not_trivial 19%) are handled.
- Plane spread (best->worst): architecture_flow 4.36 ... language_mechanics
  2.66. Target/plane choice moves ~1.7 pts, ~5x the model-size effect.
- 1.5b vs 7b: 2.97 vs 3.33, but three 1.5b runs spanned 2.90-3.17. The 7b edge
  (0.16) is inside the noise floor — NOT a proven win. Needs n>=30 to claim.
- Report: ../mergelearn-eval/runs/CONCLUSIONS.md.

## 2. Agent-authored card import (S11) — branch feat/agent-authored-cards

Design decision (user chose Option 2): the agent picks code ranges freely; the
tutor creates a synthetic concept per range so scheduling/mastery work unchanged.

Trust boundary is UNCHANGED and enforced:
- Agent's snippet text is never trusted. `freezeDraft` re-reads the cited range
  from disk (readRange) and pins it to HEAD.
- verifyFormat (deterministic) -> validateAnswerKey (optional oracle) ->
  decideStaging. Only 'active' cards attach; 'needs_review' reported, not scheduled.

Files:
- src/core/importCards.ts (new): freeze/adapt/orchestrate + createEndpointLlm.
- src/core/answerKey.ts: oracle hardened against a dead endpoint (honest skip).
- src/core/types.ts: +'agent_import' LearningItemSource.
- src/cli.ts: `cards import --file <drafts.json> [--oracle]`.
- skill/authoring/SKILL.md (new): drives the coding agent to author drafts.
- tests: importCards.test.ts (6), answerKey unreachable-oracle regression (1).

Verification: build clean, 216 tests pass (was 209). E2E against a real git
repo confirmed the full chain: draft -> frozen snippet -> card in `today`;
malformed card -> needs_review.
