## 1. What is MergeLearn Tutor

A local-first code learning tool. It reads git history from a target repository, extracts repo-specific concepts using deterministic rules and TypeScript AST analysis, and turns real diffs, files, docs, and commits into short active-recall cards.

It is:

- Local-first by default (no telemetry, no remote LLM calls, no target repo execution)
- CLI-controlled (the browser session is just a view over `.skilltrace/state.json`)
- A knowledge-debt tool (not a SaaS dashboard, not a PR blocker, not a remote AI reviewer)

## 3. Platform architecture

The browser session is organized into five primary surfaces, each backed by a route and a renderer in `src/session/server.ts`.

Primary surfaces (top nav):

1. Workbench (`/workbench`) - interactive map of local learning nodes with semantic filters and detail drawer
2. Practice (`/practice`) - focused one-card retrieval-practice loop with keyboard shortcuts
3. Map (`/map`) - unified surface with three modes: local-graph, provenance, skill-map
4. Audit (`/audit`) - consolidated quality view for cards, questions, study, and calibration
5. Setup (`/plan`) - guided learning plan wizard with inline course creation

Secondary surfaces (sub-nav, legacy routes still working):

- Review (`/`) - full queue view
- Courses (`/courses`) - advanced course form
- Questions (`/questions`) - question bank management
- Timeline (`/timeline`) - evidence timeline (legacy, now part of Map)
- Graph (`/graph`) - evidence graph (legacy, now part of Map)
- Study (`/study`) - active-control pilot
- History (`/history`) - card history (legacy, now part of Audit)
- Progress (`/progress`) - concept mastery (legacy, now part of Map)
- Preferences (`/preferences`) - question mix settings

### 5b. Agent Failure Warning: HTTP 400 Bad Request

**CRITICAL Gotcha for LLM/Agent execution:**
During long runs, truncated contexts, or agent transitions, if an assistant message block ends with a `thinking` block (or has a trailing thinking/tool block with no subsequent text block), the Claude API will reject it with:
`Error: HTTP 400 Bad Request (messages.191: The final block in an assistant message cannot be thinking)`

**Mitigation / Action for incoming agents:**
- **Always conclude your turn with a text block**. Never leave a trailing tool call or empty thinking block without a corresponding final response or summary message.
- If you notice a 400 error, check the conversation transcript files (e.g. `transcript.jsonl`) to ensure the final block has a non-empty text response.

## 8. Known issues and gotchas

### Line endings

Files written from the Windows side can get CRLF endings. If `git diff` shows massive changes with no content diff, normalize:

```bash
git diff --ignore-cr-at-eol
# If only line endings differ:
find . -name "*.md" -type f -exec sed -i 's/\r$//' {} +
git add -u
```

### Zone.Identifier files

Windows writes alternate data stream files that show up in git status. They are inert. To ignore:

```bash
echo "*Zone.Identifier" >> .gitignore
```

### Server not responding

If the demo server stops:

```bash
# Check if port 4197 is in use
ss -ltnp | grep ':4197 '

# If a process is there, kill it
kill $(ss -ltnp | sed -n 's/.*127\.0\.0\.1:4197.*pid=\([0-9]*\).*/\1/p' | head -1)

# Start fresh
cd /home/adam/mergelearn-tutor
node dist/cli.js session --repo /tmp/mergelearn-main-demo --port 4197
```

### Test time

The CLI test takes ~20 seconds. Total test run is ~22 seconds. The 81-test suite is fast enough for a verification loop.

### Demo repo state

`/tmp/mergelearn-main-demo` is a seeded demo repo. If it gets corrupted or emptied, re-seed it:

```bash
cd /tmp
rm -rf mergelearn-main-demo
mkdir mergelearn-main-demo
cd mergelearn-main-demo
git init

```
