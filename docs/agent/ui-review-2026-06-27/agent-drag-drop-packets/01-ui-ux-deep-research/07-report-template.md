# Required report template

# MergeLearn Tutor UI/UX Deep Research Report

## 1. Executive recommendation

Summarize the recommended product shape in 5-10 bullets.

## 2. Current UI diagnosis

Identify the biggest navigation, visual hierarchy, density, terminology, and interaction problems.

## 3. Page-by-page critique

For every captured page, include:

- current job of the page;
- what works;
- what is confusing;
- what to keep, merge, move, or remove.

## 4. Recommended information architecture

Propose the new top-level navigation and where every current page/function should move.

## 5. Component and visualization proposals

Include concrete components, visualizations, and interaction states. Explain why each helps learning or usability.

## 6. Reference patterns

Discuss what to borrow and avoid from Obsidian, GitLens, Anki, Linear, Duolingo, and any other relevant products.

## 7. Implementation roadmap

Provide dependency-ordered slices. Each slice should include goal, files likely touched, risk, acceptance criteria, and verification.

## 8. Risks and anti-patterns

Call out where visual polish could harm clarity, learning honesty, privacy, or implementation cost.

## 9. Final prioritized ticket list

End with 5-10 tickets in priority order.
